# 專案說明

## 啟動規則
- 每次對話開始時，**必須先閱讀 `PROGRESS.md`**，了解目前開發進度後再回應

## 使用者
- 名稱：小積木
- 語言：請全程使用**繁體中文**與小積木對話

## 專案目標
建立一個**產品監控功能**的前後端系統，包含：
- 前端介面：展示產品監控數據、狀態儀表板
- 後端服務：資料收集、處理與 API 提供

## 行為準則
- 所有回覆、說明、註解一律使用繁體中文
- 稱呼使用者為「小積木」

## 修改限制
- **每次只修改一個功能或一個檔案**，不要在同一次回覆中大量改動多個檔案
- 若預計需要修改超過 3 個檔案、或執行時間估計超過 3 分鐘，**必須先暫停**，向小積木說明：
  1. 打算做什麼（Plan 摘要）
  2. 會動到哪些檔案
  3. 預計的影響範圍
  - 等小積木確認後才動手
